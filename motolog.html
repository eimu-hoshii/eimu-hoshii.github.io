PROJECT OVERVIEW

「MOTOLOG」は、ライダーのための燃費・メンテナンス管理アプリケーションです。 Googleスプレッドシートをバックエンドデータベースとして活用し、ReactによるモダンなUIを組み合わせることで、 スマートフォンからの高速なデータ入力と、リアルタイムなデータ可視化を実現しました。

従来のGASアプリの枠を超え、SPA（Single Page Application）として動作する設計を採用。 給油履歴からの自動燃費計算ロジックを搭載し、ユーザーの手計算を完全に排除しています。

FEATURES

1. 自動燃費計算アルゴリズム
ユーザーは「オドメーター」「給油量」を入力するだけ。アプリが前回の給油記録を自動探索し、走行距離の差分から燃費(km/L)を即座に算出します。

2. ハイブリッド・データ管理
給油ログ（Fuel）とメンテナンスログ（Maintenance）を単一のタイムラインで管理。アイコンによる視覚的な区別を行い、車両の健康状態を一元管理可能です。

3. 分析ダッシュボード
Chart.jsを用いた動的なグラフ描画機能を搭載。燃費の推移を可視化し、エンジンの不調や季節要因による変動を早期に発見できます。

DEVELOPMENT NOTE

GASの `HtmlService` 上でReactを動作させる構成を採用しました。 最大の課題は、`google.script.run` による非同期通信とUIの整合性でしたが、 Reactの `useEffect` フックによる状態管理と、ローカル環境（プレビュー）用のモックデータ自動切替ロジックを実装することで、 開発効率と安定性を両立させました。

UIデザインは、夜間のライディングでも目に優しいダークテーマ（Slate-900）を基調とし、 重要な数値にはCyan（#06b6d4）のアクセントカラーを使用することで、視認性を高めています。
